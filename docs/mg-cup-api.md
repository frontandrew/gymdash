## Чемпионаты

### Список чемпионатов

```
GET /championships/list
```

#### Параметры

| Параметр   | Тип    | Обязательный | Описание |
|------------|--------|:------------:|----------|
| `signature`| string | ✓ | API-ключ |
| `types`    | string | —  | Фильтр по типу: `online`, `gp`, `offline`. Несколько значений — через запятую. По умолчанию — все типы. |
| `fromYear` | int    | —  | Начиная с какого года (4 цифры). Если не указан ни `fromYear`, ни `toYear` — возвращается только текущий год. |
| `toYear`   | int    | —  | До какого года включительно (4 цифры). |

#### Пример запроса

```
GET /championships/list?signature=<key>&types=online,gp&toYear=2022
```

#### Формат ответа

```json
[
  {
    "id": 18,
    "title": "Triangle gp",
    "year": "2021",
    "description": "<p>Описание чемпионата</p>",
    "stagesCount": 7,
    "type": "online"
  }
]
```

| Поле          | Тип    | Описание |
|---------------|--------|----------|
| `id`          | int    | ID чемпионата |
| `title`       | string | Название чемпионата |
| `year`        | string | Год проведения |
| `description` | html   | Описание чемпионата |
| `stagesCount` | int    | Количество этапов |
| `type`        | string | Тип: `online`, `offline`, `gp` |

### Информация о чемпионате

```
GET /championships/get
```

#### Параметры

| Параметр    | Тип    | Обязательный | Описание |
|-------------|--------|:------------:|----------|
| `signature` | string | ✓ | API-ключ |
| `id`        | int    | ✓ | ID чемпионата |
| `type`      | string | ✓ | Тип чемпионата: `online`, `offline`, `gp` |

#### Пример запроса

```
GET /championships/get?signature=<key>&id=1&type=gp
```

#### Формат ответа

Структура ответа зависит от типа чемпионата. Общие поля для всех типов:

| Поле          | Тип    | Описание |
|---------------|--------|----------|
| `id`          | int    | ID чемпионата |
| `title`       | string | Название чемпионата |
| `year`        | int    | Год проведения |
| `description` | html   | Описание чемпионата |
| `stages`      | array  | Список этапов (см. ниже) |

**Поля этапа — тип `gp` и `online`:**

| Поле                  | Тип    | Описание |
|-----------------------|--------|----------|
| `id`                  | int    | ID этапа |
| `status`              | string | Статус этапа¹ |
| `title`               | string | Название этапа |
| `description`         | html   | Описание этапа |
| `usersCount`          | int    | Количество активных участников |
| `class`               | string | Класс этапа² |
| `trackUrl`            | url    | Фото трассы |
| `dateStart`           | int    | Дата начала (unixtime) |
| `dateEnd`             | int    | Дата завершения (unixtime) |
| `referenceTimeSeconds`| int    | Эталонное время (мс) |
| `referenceTime`       | string | Эталонное время (`мин:сек.мсек`) |
| `bestTimeSeconds`     | int    | Лучшее время (мс) |
| `bestTime`            | string | Лучшее время (`мин:сек.мсек`) |
| `bestUserId`          | int    | ID спортсмена с лучшим результатом |
| `bestUserFirstName`   | string | Имя спортсмена с лучшим результатом |
| `bestUserLastname`    | string | Фамилия спортсмена с лучшим результатом |
| `bestUserFullName`    | string | Полное имя спортсмена с лучшим результатом |
| `bestUserCity`        | string | Город спортсмена с лучшим результатом |

> ¹ **Статусы (gp / online):** `Предстоящий этап`, `Приём результатов`, `Подведение итогов`, `Прошедший этап`, `Этап отменён`

**Дополнительные поля этапа — тип `offline`:**

| Поле        | Тип    | Описание |
|-------------|--------|----------|
| `city`      | string | Город проведения этапа |
| `dateOfThe` | int    | Дата этапа (unixtime) |

> **Статусы (offline):** `Предстоящий этап`, `Открыта регистрация на этап`, `Завершена регистрация на этап`, `Текущий этап`, `Подведение итогов`, `Прошедший этап`, `Этап отменён`

> ² **Классы этапа:** `A`, `B`, `C1`, `C2`, `C3`, `D1`, `D2`, `D3`, `D4`, `N`


#### Параметры

| Параметр    | Тип    | Обязательный | Описание |
|-------------|--------|:------------:|----------|
| `signature` | string | ✓ | API-ключ |
| `id`        | int    | ✓ | ID чемпионата |
| `type`      | string | ✓ | Тип чемпионата: `offline` или `gp` |

#### Пример запроса

```
GET /championships/results?signature=<key>&id=1&type=gp
```

#### Формат ответа

```json
[
  {
    "place": 1,
    "userId": 42,
    "userFullName": "Иван Иванов",
    "userFirstName": "Иван",
    "userLastName": "Иванов",
    "userCity": "Москва",
    "userClass": "A",
    "points": 150,
    "stages": [
      {
        "id": 1,
        "title": "Этап 1",
        "points": 75
      }
    ]
  }
]
```

| Поле           | Тип    | Описание |
|----------------|--------|----------|
| `place`        | int    | Место в чемпионате |
| `userId`       | int    | ID спортсмена |
| `userFullName` | string | Полное имя |
| `userFirstName`| string | Имя |
| `userLastName` | string | Фамилия |
| `userCity`     | string | Город |
| `userClass`    | string | Класс спортсмена |
| `points`       | int    | Суммарное количество баллов |
| `stages`       | array  | Баллы по каждому этапу |
| `stages[].id`  | int    | ID этапа |
| `stages[].title`| string| Название этапа |
| `stages[].points`| int  | Баллы за этап |

## Спортсмены

### Список спортсменов

```
GET /users/list
```

Результаты сортируются по классу от A до N, внутри класса — по фамилии.

#### Параметры

| Параметр    | Тип    | Обязательный | Описание |
|-------------|--------|:------------:|----------|
| `signature` | string | ✓ | API-ключ |
| `limit`     | int    | — | Количество записей. Максимум — 100, по умолчанию — 20. |
| `offset`    | int    | — | Смещение для пагинации. |
| `class`     | string | — | Фильтр по классу: `A`, `B`, `C1`, `C2`, `C3`, `D1`, `D2`, `D3`, `D4`, `N`. |

#### Формат ответа

```json
{
  "total": 250,
  "items": [...]
}
```

| Поле          | Тип    | Описание |
|---------------|--------|----------|
| `total`       | int    | Всего найдено результатов |
| `items`       | array  | Массив спортсменов |

**Поля спортсмена:**

| Поле          | Тип    | Описание |
|---------------|--------|----------|
| `id`          | int    | ID спортсмена |
| `fullName`    | string | Полное имя |
| `firstName`   | string | Имя |
| `lastName`    | string | Фамилия |
| `city`        | string | Город |
| `country`     | string | Страна |
| `class`       | string | Класс спортсмена |
| `motorcycles` | array  | Список мотоциклов |
| `imgUrl`      | string | Фото спортсмена |
| `number`      | int    | Номер спортсмена (null по умолчанию) |

## Калькулятор

```
GET /calculate/get
```

Рассчитывает, в какие классы попадает время спортсмена относительно эталона.

#### Параметры

| Параметр    | Тип    | Обязательный | Описание |
|-------------|--------|:------------:|----------|
| `signature` | string | ✓ | API-ключ |
| `class`     | string | ✓ | Класс спортсмена — ID или буквенное обозначение на русском или английском, регистр не важен. |
| `time`      | string | ✓ | Время спортсмена (см. форматы ниже). |

**Поддерживаемые форматы времени:**

| Формат           | Пример       |
|------------------|--------------|
| `mm:ss.msec`     | `01:21.540`  |
| `mm:ss,msec`     | `01:21,540`  |
| `mm:ss:msec`     | `01:21:540`  |
| `mm:ss`          | `01:21`      |
| `mm`             | `01`         |

> Минуты и секунды не могут превышать 60. Миллисекунды округляются до 3 знаков.

#### Примеры корректных запросов

```
GET /calculate/get?signature=<key>&class=д1&time=01:21,54
GET /calculate/get?signature=<key>&class=d1&time=01:21:54
GET /calculate/get?signature=<key>&class=D1&time=01:21.54
GET /calculate/get?signature=<key>&class=d1&time=01:21
```

#### Примеры некорректных запросов

```
GET /calculate/get?class=д1&time=01,21   // неверный разделитель
GET /calculate/get?class=д1&time=01.21   // неверный разделитель
GET /calculate/get?class=дд&time=01:21   // несуществующий класс
GET /calculate/get?class=д1&time=01:555  // секунды > 60
```

#### Формат ответа

```json
{
  "referenceTime": "01:10.000",
  "results": [
    {
      "class": "A",
      "classPercent": "80%",
      "startTime": "00:56.000",
      "endTime": "01:10.000"
    }
  ]
}
```

| Поле                    | Тип    | Описание |
|-------------------------|--------|----------|
| `referenceTime`         | string | Эталонное время (`мин:сек.мсек`) |
| `results`               | array  | Границы классов |
| `results[].class`       | string | Класс |
| `results[].classPercent`| string | С какого процента начинается класс |
| `results[].startTime`   | string | Нижняя граница попадания в класс |
| `results[].endTime`     | string | Верхняя граница попадания в класс |
